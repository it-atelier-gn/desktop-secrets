Resolve secrets,

DesktopSecrets is a resolution engine for .env.tpl templates. It bridges your code and your KeePass vaults — resolving secret references into environment variables so credentials never end up in source files.

Features

The bridge between your code and your vaults.

DesktopSecrets doesn't store secrets — it resolves them from the tools that do.

🔑

KeePass vault lookups

Read entries from .kdbx vaults using flexible path matching — wildcards, absolute paths, attribute selection. Your vault stays in charge.

📄

Template-based .env files

Write .env.tpl files with secret references instead of plaintext credentials. Commit them safely — tplenv resolves them at runtime.

🔁

Recursive aliases & chaining

Define vault shortcuts in aliases.yaml. Chain master passwords across vaults — one unlock can feed into the next.

⏳

Caching daemon

A background daemon caches unlocked vaults for a configurable TTL. No repeated master password prompts during your workflow.

🖥️

System tray & GUI prompts

Runs in the system tray with settings UI. Master passwords and user-provided secrets are entered via native dialogs.

📦

CLI & Go library

Use tplenv to resolve entire templates, getsec for single values, or import as a Go module for programmatic resolution.

How it works

Three steps.

From template to resolved environment in seconds.

01

Create a .env.tpl

Replace plaintext credentials with secret references like keepass(vault.kdbx|api-key). Commit the template safely to git.

02

Run tplenv or getsec

The CLI connects to the background daemon, which unlocks your KeePass vault (prompting only when needed) and resolves every reference.

03

Secrets flow into your process

Use tplenv run to launch a command with resolved env vars injected, or pipe the output wherever you need it.

Syntax

Simple references. Powerful resolution.

A compact expression language for retrieving secrets from any provider.

Secret references — .env.tpl
# KeePass vault entry
API_KEY=keepass(vault.kdbx|/AWS/Prod/api-key)

# Wildcard matching
DB_PASS=keepass(vault.kdbx|/DB/**/password)

# Attribute selection
DB_USER=keepass(vault.kdbx|/DB/prod|UserName)

# User prompt
TOKEN=user(Enter deploy token)
Aliases — aliases.yaml
# Simple file alias
personal: C:\Vaults\personal.kdbx

# Alias with chained master password
cloud:
  file: C:\Vaults\cloud.kdbx
  master: keepass(&personal|Cloud Master)

# Usage
SECRET=keepass(&cloud|/Prod/api-key)
